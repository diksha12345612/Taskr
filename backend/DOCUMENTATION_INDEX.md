## 💾 Schema Summary

### Models (3)
- **User** - Admin or Member (name, email unique, password, createdAt)
- **Project** - Active/Review/Hold (name, description, progress 0-100, createdAt)
- **Task** - Todo/InProgress/Completed | High/Medium/Low (title, description, priority, status, dueDate, createdAt, belongs to Project & User)

### Enums (4)
- **UserRole**: Admin, Member
- **ProjectStatus**: Active, Review, Hold
- **TaskStatus**: Todo, InProgress, Completed
- **TaskPriority**: High, Medium, Low

### Relationships
- User → Task (one user, many tasks, setNull on delete)
- Project → Task (one project, many tasks, cascade on delete)

## 🛠️ Essential Commands

```bash
# Setup
npm run prisma:generate         # Generate Prisma Client
npm run migrate                 # Create database tables

# Development
npx prisma studio             # Visual database browser
npm run db:push               # Push schema directly (dev)
npm run db:reset              # Clear database (dev only!)

# Production
npm run migrate:prod          # Apply migrations safely

# Troubleshooting
npx prisma migrate status     # Check status
npx prisma validate           # Validate schema
```
